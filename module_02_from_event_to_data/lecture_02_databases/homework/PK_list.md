Список первичных ключей (Primary Keys, PK) для всех сущностей:

"Категории" ("Categories") - category_id
"Города" ("Cities") - city_id
"Страны" ("Countries") - country_id
"Покупатели" ("Customers") - customer_id
"Сотрудники" ("Employees") - employee_id
"Товары" ("Products") - product_id
"Продажи" ("Sales") - sales_id
"Филиалы" ("Shops") - shop_id

Для всех перечисленных сущностей выбор таких PK оправдан тем, что они уникально идентифицируют все имеющиеся записи в таблицах дынных, так как значения в них являются последовательными целыми числами, генерируемыми по мере добавления записей в таблицы.

Например, каждому покупателю при регистрации присваивается собственный уникальный номер, записываемый в customer_id - он используется для связи покупателя с заказами, с его обращениями и платежами, а категориям такие уникальные номера присвоены при формировании ассортимента магазина.

Второй причиной использования вышеперечисленных PK это тип данных тех значений, которые хранятся в записях - это целые числа. Использование такого типа данных эффективнее.
