Для сущностей sales и products определите следующие ограничения (которые необходимо будет реализовать при создании таблиц):

1.Укажите 3–4 атрибута, которые не могут быть пустыми (NOT NULL), и обоснуйте свой выбор.
2. Укажите, какие ID должны быть уникальными (UNIQUE).
3. Предложите 2–3 ограничения CHECK (например, для числовых полей, которые не могут быть отрицательными, или для логических полей, которые могут принимать только два значения).

Ограничения.


Ограничение NOT NULL.

Ограничение NOT NULL следует использовать для атрибутов первичных и внешних ключей, а также тех атрибутов, без значений которых будет утрачен смысл таблицы-отношения.
 
В таблице-отношении "Товары" ("Products") ограничение NOT NULL следует ввести для атрибутов: product_id (первичный ключ), category_id (внешний ключ), product_name и price (без этих двух атрибутов будет утрачен общий смысл таблицы-отношения).

В таблице-отношении "Продажи" ("Sales") ограничение NOT NULL следует ввести для атрибутов: sales_id (первичный ключ); employee_id, customer_id, product_id (внешние ключи); а также для quantity и discount (без этих атрибутов будет невозможно определить общую стоимость заказа total_price).


Ограничение UNIQUE.

Ограничение UNIQUE гарантирует, что каждое значение атрибута будет уникальным, но при этом допускает NULL значения.

В таблице-отношении "Товары" ("Products") ограничение UNIQUE стоит ввести для атрибута product_name, чтобы исключить вероятность повторного добавления уже имеющегося в таблице-отношении продукта.

В таблице-отношении "Продажи" ("Sales") ограничение UNIQUE не требуется. Оно могло бы быть использовано для атрибута transaction_number, чтобы гарантировать уникальность номеров для каждого заказа, однако, номера транзакций определяются автоматически на основе первичного ключа sales_id, поэтому введение такого ограничения избыточно.

Ограничение CHECK.

Ограничение CHECK позволяет определить правило или условие, которое должно быть истинным для каждого значения в определенном столбце.

В таблице-отношении "Товары" ("Products") ограничение CHECK стоит ввести для атрибутов: price (>=0), vitality_days (>=1), resistant ('Yes', 'No'), is_allergic ('Yes', 'No').

В таблице-отношении "Продажи" ("Sales") ограничение CHECK стоит ввести для атрибутов: quantity (>=1) и discount (>=0, <=1).
