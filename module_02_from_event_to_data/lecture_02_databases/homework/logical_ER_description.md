Типы связей между сущностями:

a. "Продажи" ("Sales") M:1 "Товары" ("Products") - каждый из товаров (например овсяное молоко) со своим уникальным product_id может быть продан множество раз - и каждой из таких продаж будет соответствовать новая уникальная запись sales_id.
b. "Сотрудники" ("Employees") M:1 "Города" ("Cities") - несколько сотрудников, каждому из которых соответсвует свой уникальный employee_id, могут работать в одном городе, которому соответствует одна запись city_id (в рассматриваемой ситуации один сотрудник не может работать одновременно сразу в нескольких городах).
c. "Города" ("Cities") M:1 "Страны" ("Countries") - несколько городов с соответствующими уникальными city_id могут находиться в одной стране, которой соответствует одна запись country_id в таблице "Страны".
